Pangool User Guide

4 - Named outputs

Named outputs in Pangool are somewhat based on “MultipleOutputs” in Hadoop, but they have some differences. We found that “MultipleOutputs” has some incompatibilities, for instance it doesn’t work with output formats that write to arbitrary file names. For that purpose, we made Pangool’s named outputs write to subfolders. This way, output format implementations are free to write to any file name.

Output formats in Pangool are instances. This allowed us to implement a TupleOutputFormat that takes a Schema as a parameter. But you don't need to care about this. To add a named output that will write a Tuples with a certain Schema, all you need to do is, for instance: 

	Schema baseSchema = new Schema("schema",Fields.parse("name:string, money:int, country:string"));
	builder.addNamedTupleOutput(TUPLEOUTPUT, baseSchema);

Still, if you want to use classes instead of instances, you can wrap static classes by using the HadoopOutputFormat() class. For example: 

	builder.setOutput(new Path(OUTPUT), new HadoopOutputFormat(SequenceFileOutputFormat.class), DoubleWritable.class, NullWritable.class);
