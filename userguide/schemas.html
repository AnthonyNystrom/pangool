Pangool User Guide

Tuples & Schemas

A Pangool Tuple is an ordered list of Objects with an associated Schema. Schemas have an associated name and a list of Fields.

We can create an Schema in a strong-typed fashion: 

	List fields = new ArrayList();
	fields.add(Field.create("number1", Type.INT));
	fields.add(Field.create("string1", Type.STRING));
	fields.add(Field.create("string2", Type.STRING));
	Schema schema1 = new Schema("test1", fields);

Or we can also create it leveraging the Fields.parse() method: 

	new Schema("schema2", Fields.parse("a:int, b:string, c:string, d:string"));

Although Pangool does not provide a serialization framework by itself, it has some primitive types implemented for convenience:

Type.INTSerialized as an VInt
Type.LONGSerialized as an VLong
Type.FLOATSerialized as 4-bytes float type
Type.DOUBLESerialized as 8-bytes double type
Type.UTF8Serialized like Hadoopâ€™s Text
Type.BOOLEANSerialized as 1-byte boolean type
Type.ENUMSerialized as Vint values

For non-primitive types you can use Type.OBJECT and set arbitrary objects. These will be serialized using the Hadoop Serialization library (see link-goes-here ). So if you are using your own custom data types, you need to configure Hadoop accordingly.

Tuples meet the ITuple interface and can be instantiated via the Tuple class. You can set tuple values by field name or field index.

	ITuple tuple = new Tuple(schema1);
	tuple.set("number1", 35);
	tuple.set("string1", "foo");
